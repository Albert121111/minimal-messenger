# Minimal Messenger

Небольшой веб‑мессенджер на Supabase: регистрация с юзернеймом, поиск людей и личные диалоги с сохранением истории между устройствами.

## Запуск

1. Создайте проект в [Supabase](https://supabase.com/dashboard), откройте **SQL Editor** и выполните содержимое `supabase/schema.sql`.
2. Подтверждение email можно как включить, так и выключить: профиль создаётся серверным триггером вместе с аккаунтом, поэтому частичная регистрация не сломает вход.
3. Откройте `app.js` и вставьте `SUPABASE_URL` и `SUPABASE_ANON_KEY` из **Project Settings → API**.
4. Откройте `index.html` в браузере для локальной проверки либо опубликуйте три файла `index.html`, `styles.css`, `app.js` на любом статическом хостинге.

## Публикация

Подходит Netlify Drop, Cloudflare Pages или Vercel: это статический сайт, сервер не нужен. Данные, аккаунты и сообщения живут в Supabase, поэтому доступны после входа с любого устройства.

Для уже созданной базы сначала примените `supabase/migrations/2026-07-20-chat-hardening.sql`, затем `supabase/migrations/2026-07-21-profile-status.sql`: они исправляют правила доступа к диалогам, регистрацию, дублирование чатов при одновременном клике и добавляют статусы профиля.

Никогда не добавляйте в `app.js` ключ `service_role`: там должен быть только публичный ключ клиента.
